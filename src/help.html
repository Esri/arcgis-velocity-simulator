ArcGIS Velocity Simulator Help

Overview

Cross-platform desktop application for simulating data streams from CSV files over TCP, UDP, HTTP/HTTPS, WebSocket, and gRPC protocols. Features gesture and voice controls for hands-free operation.

Quick Start

  1. Select CSV file
  2. Choose protocol mode (TCP/UDP/HTTP/WebSocket/gRPC Server/Client)
  3. Set IP address and port
  4. Configure streaming rate
  5. Connect and Play

Connection Modes

  • TCP Server: Listen for incoming TCP connections from clients
  • TCP Client: Connect to a remote TCP server
  • UDP Server: Listen for incoming UDP datagrams
  • UDP Client: Send UDP datagrams to a remote server
  • HTTP Client: Send data via HTTP/HTTPS POST requests to a remote endpoint (e.g. an ArcGIS Velocity HTTP Receiver feed). Supports JSON, Delimited (CSV), Esri JSON, GeoJSON, and XML formats.
  • HTTP Server: Start a local HTTP/HTTPS server that accepts POST requests from clients. Use for testing or when the Velocity HTTP Receiver connects to this app.
  • WebSocket Client: Connect to a remote WebSocket server (ws:// or wss://) and send streaming data. Supports JSON, Delimited (CSV), Esri JSON, GeoJSON, and XML formats. Can send a subscription message on connect and optionally ignore the first message.
  • WebSocket Server: Start a local WebSocket server (ws:// or wss://) that accepts incoming connections and broadcasts data to all connected clients. Supports the same 5 formats as the WebSocket client.
  • gRPC Client: Connect to a remote gRPC server using HTTP/2. Supports Protobuf, Kryo, and Text serialization with client-streaming or unary RPCs.
  • gRPC Server: Start a local gRPC server that accepts incoming RPC calls from clients.

HTTP Options

  • Format: Controls the Content-Type header — JSON (application/json), Delimited/CSV (text/plain), Esri JSON (application/json), GeoJSON (application/geo+json), or XML (application/xml). Must match the format configured in the ArcGIS Velocity HTTP Receiver feed.
  • Use TLS: Enable HTTPS (port 8443 by default). Uses the OS certificate store automatically in client mode. Server mode requires a certificate and key.
  • CA cert: Custom CA certificate file (PEM). Leave empty to use the OS certificate store. Only needed for enterprise or self-signed CAs.
  • TLS cert: Client or server certificate file (PEM). Required for server-mode TLS; only needed in client mode for mutual TLS (mTLS).
  • TLS key: Private key file (PEM). Required for server-mode TLS and client-side mTLS.
  • HTTP path: URL path appended after host:port (default /). In server mode, only POST requests matching this path are accepted. In client mode, this path is used in outgoing POST URLs. Set to the Velocity feed's system-generated path (e.g. /receiver/feed-id) when connecting to a real endpoint.

WebSocket Options

  • Format: Controls the data format — JSON (application/json), Delimited/CSV (text/plain), Esri JSON (application/json), GeoJSON (application/geo+json), or XML (application/xml).
  • Use TLS: Enable WSS (WebSocket Secure, port 8443 by default). Uses the OS certificate store automatically in client mode. Server mode requires a certificate and key.
  • CA cert: Custom CA certificate file (PEM). Leave empty to use the OS certificate store. Only needed for enterprise or self-signed CAs.
  • TLS cert: Client or server certificate file (PEM). Required for server-mode TLS; only needed in client mode for mutual TLS (mTLS).
  • TLS key: Private key file (PEM). Required for server-mode TLS and client-side mTLS.
  • WS path: URL path appended after host:port (default /). In server mode, only WebSocket upgrade requests matching this path are accepted. In client mode, this path is used in the outgoing connection URL.
  • Subscription message: Optional text message sent to the server immediately after the WebSocket connection is established. Useful for subscribing to a specific data feed or channel.
  • Ignore first message: When enabled, the first message received after connecting is silently discarded. Useful when the server sends an initial handshake or acknowledgement message.
  • Custom headers: Optional JSON object of custom HTTP headers to include in the WebSocket upgrade request (client mode only). Example: {"Authorization": "Bearer token123"}.

gRPC Options

  • Serialization: Protobuf (default, external GrpcFeed protocol), Kryo (internal GrpcFeatureService, binary), or Text (internal GrpcFeatureService, UTF-8 CSV).
  • RPC Type: Client Streaming (persistent stream, higher throughput) or Unary (individual request/response per message).
  • Use TLS: Enable TLS for the gRPC connection. Uses the OS certificate store automatically.
  • Header Path Key / Path: Custom gRPC metadata header for endpoint routing (client mode only).

Controls

  • File: Select CSV file to stream
  • Connection: Set IP address and port
  • Rate: Lines per interval and timing (ms)
  • Connect/Disconnect: Establish/terminate connection
  • Play/Pause/Step: Control data streaming
  • Continuous Loop: ➿ button toggles file looping

Interface

  • Line Preview: Shows current line content
  • Navigation Slider: Scrub through data file
  • Status Bar: Shows position, state, and lines sent
  • View Toggle: ⇔ button switches full/compact view
  • Status Log Toggle: 📜 button shows/hides log panel

Gesture Controls

  • Setup: Click 📷 button, allow webcam access
  • Gestures:
  • 👍 Thumbs Up → Connect
  • 🤙 Pinky Up → Disconnect
  • 👊 Closed Fist → Play/Resume
  • 🖐️ Open Palm → Pause
  • ✌️ Victory Sign → Step
  • Toggle Display: 📄 button shows/hides gesture feedback

Voice Controls

  • Setup: Click 🎤 button, allow microphone access
  • Requirements: An active internet connection is required for voice commands. If the connection fails, the system will attempt to reconnect automatically.
  • Commands:
  • "connect" / "disconnect"
  • "play" / "start" / "pause" / "stop"
  • "step"
  • "switch" / "toggle view"

Status & Logging

  • Status Bar: Bottom bar shows app state with emoji indicators
  • 🔴 Disconnected
  • 🟢 Connected - Ready
  • ▶️ Connected - Playing
  • ⏸️ Connected - Paused
  • 👣 Stepped (temporary)
  • Line Counter: Shows current position (e.g., "Line 10 of 100")
  • Lines Sent: Tracks total lines sent in session
  • Log Controls: 🖐️ and 🎤 buttons toggle gesture/voice logging
  • Clear Log: 🗑️ button clears status messages
  • Sort Order: Use the new sort button to toggle between Ascending and Descending order (default: Ascending). The icon updates to reflect the current order.

Customization

  • 15 Themes: 🔵 Blue, 🟡 Color Blind, 🌙 Dark, 🌫️ Dark Gray, 🟢 Green, ⚫ High Contrast, ☀️ Light, ☁️ Light Gray, 🌌 Midnight, ☕ Mocha, 🌊 Ocean, 🌸 Rose, 🌺 Rose Dark, 🌅 Sunset, 💻 System
  • Window Opacity: 11 levels from 50% to 100% transparency
  • Font Options: Size (6px-25px) and family customization
  • Auto-Save: All settings persist between sessions
  • Context Menu: Right-click for theme, opacity, and config options

Command Line Interface Dialog

The interactive command-line reference now lives in its own Command Line Interface dialog, separate from this product Help dialog.

  • Open it from the keyboard: Press F3 from the main window.
  • Open it from the menu: Use Help → Command Line Interface.
  • Open it from the context menu: Right-click in the main window and choose Command Line Interface.
  • Use it for automation help: Browse searchable and sortable parameters, supported values, defaults, examples, and headless notes.
  • Filter and refine: Use quick category chips, active filter pills, and sticky sortable columns to narrow the parameter table.
  • Copy or export: Copy example commands or export the currently visible rows as TSV, CSV, Markdown, or JSON.
  • Keyboard shortcuts: In that dialog, use Ctrl/Cmd + F or / to focus the filter, and Escape to close it.
